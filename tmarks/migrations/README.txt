数据库迁移文件说明
================================================================================

文件列表
--------------------------------------------------------------------------------

1. d1_console_pure.sql
   用途: 新用户一键创建所有表和索引
   使用: 在 Cloudflare D1 控制台直接执行
   场景: 第一次部署项目

2. update_for_existing_users.sql
   用途: 老用户增量更新（添加新字段和索引）
   使用: 在 Cloudflare D1 控制台直接执行
   场景: 从 GitHub 拉取新代码后
   包含: tag_layout 字段、sort_by 字段、6 个性能优化索引
   注意: 如果报错 "duplicate column name" 或 "index already exists"，说明已存在，可以忽略

3. 0003_add_general_settings.sql
   用途: 添加通用设置字段（搜索和标签自动清空）
   使用: 自动迁移系统会自动执行，或在 D1 控制台手动执行
   场景: 从 GitHub 拉取新代码后
   包含: 4 个通用设置字段
   注意: 使用自动迁移系统可避免重复执行

4. drop_all_tables.sql
   用途: 删除所有表（危险操作）
   使用: 在 Cloudflare D1 控制台直接执行
   场景: 完全重置数据库
   警告: 会丢失所有数据！

5. full_schema.sql
   用途: 完整数据库结构文档
   使用: 仅供参考，不要在控制台执行
   场景: 查看表结构、索引定义


使用指南
================================================================================

🎯 推荐方式：自动迁移系统（新功能）
--------------------------------------------------------------------------------
本项目已配置自动迁移系统，可以自动检测和执行新的迁移文件。

详细文档: 数据库迁移自动化指南.md

快速使用:
  本地开发: pnpm db:auto-migrate:local
  生产环境: pnpm db:auto-migrate

特点:
  ✅ 自动检测新迁移
  ✅ 避免重复执行
  ✅ 记录迁移历史
  ✅ Git pull 后自动提示


📝 传统方式：手动执行
--------------------------------------------------------------------------------

新用户（第一次部署）
  执行: d1_console_pure.sql

老用户（拉取新代码后）
  方式1: 使用自动迁移（推荐）
    pnpm db:auto-migrate:local
  
  方式2: 手动执行
    执行: update_for_existing_users.sql
    然后: 0003_add_general_settings.sql

完全重置（删除所有数据）
  1. 执行 drop_all_tables.sql
  2. 执行 d1_console_pure.sql
